## Design Overview

### Design Philosophy

**"Hidden Complexity"** - Simple surface, powerful engine.

The Results Roofing experience is designed to feel as easy as booking a hotel room while handling the complexity of a $15,000+ home improvement purchase. Users should feel:

1. **In control** - Never pressured, always informed
2. **Confident** - Trust signals visible, transparent pricing
3. **Efficient** - 10 minutes from address to signed contract
4. **Supported** - Help is available without requiring a phone call

**Design Principles:**

| Principle | What It Means | Example |
|-----------|--------------|---------|
| Progressive Disclosure | Show only what's needed now | Financing shown after package selection, not before |
| Clear Wayfinding | User always knows where they are | Persistent progress indicator in checkout |
| Instant Feedback | Every action gets immediate response | Address autocomplete, loading states |
| Mobile-First | 54% decide within 4 hours, mostly on phone | Touch targets, thumb-friendly CTAs |
| Trust Through Transparency | Show the work, explain the price | Itemized breakdowns, credential badges |

**Visual Inspiration:**

- **Dune (2021):** Earth tones, sand, tactile warmth, timeless weight
- **OpenAI:** Clean minimalism, generous whitespace, quiet confidence, sophisticated typography

**What This Conveys:**

- Trust and stability (not a fly-by-night contractor)
- Premium quality (you get what you pay for)
- Modern and tech-forward (we're different)
- Regional authenticity (warm palette fits premium markets in TX, GA, NC, AZ)

### Platform Targets

| Platform | Considerations |
|----------|----------------|
| **Mobile Web (Primary)** | iOS Safari, Chrome Android; touch-first design; 320px min width; responsive to 428px typical; thumb-zone navigation; no horizontal scroll |
| **Tablet Web** | iPad Safari, Android tablets; 768px breakpoint; hybrid touch/pointer; landscape and portrait support |
| **Desktop Web** | Chrome, Firefox, Safari, Edge; 1024px+ breakpoint; mouse/keyboard interaction; wider layouts with sidebar options |
| **Progressive Web App** | Installable on mobile; offline quote viewing (P1); home screen icon |

**Browser Support:**

| Browser | Minimum Version | Market Share (Target Markets) |
|---------|-----------------|---------------------|
| Chrome | 90+ | ~65% |
| Safari | 14+ | ~25% |
| Firefox | 88+ | ~5% |
| Edge | 90+ | ~5% |

**Not Supported:** IE11, Opera Mini, UC Browser

## Interaction Patterns

### Input Methods

| Input Method | Supported | Key Considerations |
|--------------|-----------|-------------------|
| Mouse/Pointer | Yes | Hover states on cards, buttons; drag not used |
| Touch | Yes | 44x44px minimum tap targets; swipe for mobile carousels |
| Keyboard | Yes | Full tab navigation; Enter to submit forms; Escape to close modals |

### Form Interactions

**Text Inputs:**
- Focus: Border changes to Sandstone, subtle glow
- Error: Red border, inline error message below
- Success: Green checkmark icon (for validated fields like address)
- Disabled: Gray background, reduced opacity

**Radio/Checkbox:**
- Custom styled (Ark UI)
- Clear selected state
- Grouped options use cards for easier touch targeting

**Buttons:**
- Primary: Sandstone background, white text
- Hover: Terracotta background
- Active: Slight scale down (95%)
- Disabled: Gray, reduced opacity, no pointer events
- Loading: Spinner replaces text, button disabled

### Keyboard Shortcuts

Minimal shortcuts for this consumer app:

| Category | Action | Shortcut |

| General | Submit form | Enter (when in form) |

## Responsive Design

### Breakpoint Strategy

**Approach:** Mobile-first

Start with mobile layout (320px), add complexity for larger screens.

| Breakpoint | Width Range | Layout Changes |
|------------|-------------|----------------|
| **Mobile** | 0 - 767px | Single column; stacked cards; bottom sticky CTAs; hamburger menu (if needed) |
| **Tablet** | 768px - 1023px | Two-column cards; more horizontal spacing; sidebar optional |
| **Desktop** | 1024px+ | Full three-column comparison; sidebar checkout summary; horizontal nav |

### Responsive Behaviors

**Navigation:**
- Mobile: Logo + minimal header; primary actions in sticky footer
- Tablet: Same as mobile, more horizontal breathing room
- Desktop: Full header with trust badges; no sticky footer

**Package Cards:**
- Mobile: Full-width, stacked vertically, swipeable optional
- Tablet: Two cards per row, "Best" below
- Desktop: Three cards side-by-side, equal height

**Checkout Summary:**
- Mobile: Collapsible bar at bottom, above CTA
- Tablet: Same as mobile
- Desktop: Fixed sidebar (right side, 320px width)

**Portal Navigation:**
- Mobile: Bottom tab bar (5 items)
- Tablet: Bottom tab bar
- Desktop: Side navigation (left, 240px width)

**Forms:**
- Mobile: Full-width inputs, stacked labels
- Desktop: Inline labels where appropriate, max-width 480px

### Minimum Supported Dimensions

- **Minimum width:** 320px (iPhone SE)
- **Minimum height:** 568px (iPhone SE)
- **Rationale:** Oldest supported modern iPhone; smaller devices are negligible market share

## Animation and Transitions

### Animation Philosophy

**Approach:** Subtle and functional

Animations should:
1. Provide feedback (button pressed, action completed)
2. Guide attention (new content appearing)
3. Smooth transitions (between states)

Animations should NOT:
1. Delay the user
2. Distract from content
3. Cause motion sickness

**Reduced Motion:** All animations respect `prefers-reduced-motion`. When enabled:
- Transitions become instant
- Loading spinners remain (functional, not decorative)
- Fade effects removed

### Transition Types

| Transition | Duration | Easing | Usage |
|------------|----------|--------|-------|
| Page/route transition | 150ms | ease-out | Between quote steps |
| Modal open | 200ms | ease-out | All modals |
| Modal close | 150ms | ease-in | All modals |
| Hover state | 150ms | ease | Buttons, cards |
| Focus state | 0ms | - | Immediate (accessibility) |
| Accordion expand | 200ms | ease-out | FAQ, breakdowns |
| Toast appear | 200ms | ease-out | Notifications |
| Toast disappear | 150ms | ease-in | Auto-dismiss |

### Loading States

| Context | Loading Pattern | Duration Threshold |
|---------|----------------|-------------------|
| Page load | Skeleton screens | Immediate |
| Data fetch (quotes) | Skeleton cards | Immediate |
| Address validation | Inline spinner | Immediate |
| Payment processing | Button spinner + disabled form | Immediate |
| Roofr measurement pending | Progress message + illustration | Show status, not spinner |

**Skeleton Screens:**
- Used for quote cards, portal content
- Animate with subtle shimmer (pulse)
- Match exact layout of loaded content

**Spinners:**
- Used for inline operations (address lookup, form submit)
- Sandstone color
- 20px for inline, 32px for buttons, 48px for page-level

### Micro-interactions

| Element | Interaction | Animation |
|---------|-------------|-----------|
| Button | Press | Scale to 95%, 100ms |
| Card | Hover | Shadow increases, 150ms |
| Checkbox | Check | Checkmark draws in, 100ms |
| Progress step | Complete | Fill color, checkmark appears, 200ms |
| Confirmation | Success | Checkmark scales up with bounce, 300ms |
| Error | Shake | Subtle horizontal shake, 200ms |
